Summary of SOLID Principles

1.Introduction
--> SOLID Principles are the principles used for the following purposes:-
easy to debug
easy to test
easy to run
readable code


-->Means in short the SOLID Principles are used to make the good class which is easy to understand and readable.

Following is the Rules for writing a good class Code:
S-->Single Responsibility Principle
O->Open/Closed Principle
L-->Liskov Substitution Principle
I-->Interface Segregation Principle
D-->Dependency Inversion Principle

4.L-->Liskov Substitution Principle
-->here in this it states that the child class should be behave same as the parent class (child class should behave same as parent class).
-->eg . as Car is parent class and if child class like Toyota ,BMW should be behave as same as parent class.



class Car:
	def __init__(self,name):
		self.name=name

	def brake():
		print("Brake")

class Toyota(Car):
	def brake():
		print("Brake")

t = Toyota()
print(t.brake())

5.I-->Interface Segregation
-->in this it states that all related method do not put in one class as it will make the big interface distribute among the small interface (Segregation into smaller interface)
-->eg:- suppose in class student printdocument,printphoto,study,marksheet etc method are there so distribute it among the other interface(class)



Example Code
class Student:
	def __init__(self,name):
		self.name=name

	def printdocument()
		print("Print Document")
	def printphoto()
		print("Print Photo")
	def marksheet()
		print("Marksheet")


#so here a large interface is made so it is divided into small interface or class

class PrintDocument:
	def printdocument():
		print("PrintDocument")
	def printphoto():
		print("PrintPhoto")

class Marksheet():
	def marksheet():
		print("Marksheet")



p = PrintDocument()
print(p.printdocument())
